Jawaban
1. Perbedaan antara 2 method tersebut adalah cara pemecahan masalahnya; pangkatBF() akan memecahkan pangkat dengan satu perulangan utuh (iterasi) untuk mengalikan basis (a) sebanyak jumlah pangkat (n). Sedangkan pangkatDC() akan membagi nilai pangkat (n) menjadi dua (dibagi 2) secara rekursif, sehingga meminimalisir langkah perkalian karena nilai kalkulasi setengah pangkat sebelah kiri dikalikan dengan setengah kalkulasi sebelah kanannya. 

2. Ya, tahap combine sudah termasuk ke dalam kode tersebut. Proses combine terjadi pada bagian return di dalam kondisi else ketika fungsi menggabungkan potongan kecil hasil conquer-nya. Khususnya pada baris ini: return (pangkatDC(a, n/2) * pangkatDC(a, n/2) * a); dan return (pangkatDC(a, n/2) * pangkatDC(a, n/2));. 

3. Pada dasarnya parameter sudah tidak terlalu relevan (bisa dihilangkan), karena kelas Pangkat sudah menyimpan dan mendeklarasikan atribut nilai dan pangkat di tingkat class lewat perantara konstruktor. Jika method pangkatBF() dibuat tanpa parameter, maka pengaksesan nilainya bisa langsung memanggil atribut instansnya seperti berikut: 
```
int pangkatBF() {
    int hasil = 1;
    for (int i = 0; i < pangkat; i++) {
        hasil = hasil * nilai;
    }
    return hasil;
}
```

4. Kesimpulan: Secara cara kerja algoritma, pangkatBF() berjalan linear mengalikan nilai a sebanyak n proses tanpa pembagian sub-masalah. Sebaliknya, pangkatDC() membelah ruang kerja dengan cara membagi dua (n/2) nilai kepangkatannya terlebih dahulu secara berulang hingga tersisa n=1. Pembagian ini memungkinkan langkah komputasi berjalan secara logaritmik dan umumnya menjadi jauh lebih cepat.
